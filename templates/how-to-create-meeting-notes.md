## 회의록의 가치

### 회의록은 무엇인가?

회의록은 단순한 기록이 아닙니다.

> "회의록은 조직의 기억 장치이자, 의사결정의 연속성을 보장하는 도구입니다."

### 왜 회의록이 필요한가?

| 목적 | 설명 |
|------|------|
| **맥락 보존** | 왜 그 결정을 했는지, 어떤 논의가 있었는지 보존 |
| **책임 명확화** | 누가 무엇을 해야 하는지 명확히 |
| **기술 전파** | 불참자도 회의 내용 이해 가능 |
| **조직 학습** | 과거 결정과 논의를 통해 패턴 학습 |
| **시간 절약** | 같은 논의 반복 방지 |

### 전문가의 통찰

**Jill Larkin (Cognitive Science)**:
> "계층적 구조가 있을 때 사람들이 정보를 2배 더 빨리 이해합니다. 회의록의 가치는 길이가 아니라 구조의 명확성에 있습니다."

**David Allen (GTD)**:
> "훌륭한 회의록이라도 '다음에 누가 무엇을 해야 하는가'가 명확하지 않으면 쓸모가 없습니다."

## Core Practice 3가지

전문가들이 강조한 절대 빼면 안 되는 핵심 원칙입니다.

---

### 1. 섹션 분리

```
┌─────────────────────────────────┐
│  결정 사항  (Decisions)          │ ← 무엇으로 정해졌는가?
│  ───────────────────────────   │
│  논의 내용  (Discussion)         │ ← 어떻게 논의되었는가?
│  ───────────────────────────   │
│  액션 아이템 (Action Items)      │ ← 누가 무엇을 해야 하는가?
└─────────────────────────────────┘
```

**근거**: [Strong] Larkin & Simon (1987) — 계층적 구조가 문제 해결 속도를 2배 이상 향상

**왜 필요한가?**
- 정보가 섞여 있으면 인지 부하가 높아짐
- 빠르게 필요한 정보를 찾을 수 있음
- 각 섹션의 목적이 명확해져 작성하기도 편해짐

---

### 2. 행위자 명시

```
❌ "다음 주까지 계획 수립"          ← 누가?
✅ "다음 주까지 @철수가 계획 수립"   ← 명확!
```

**근거**: [Expert] David Allen — 담당자가 없으면 실행되지 않음

**AAA 트리플 (Actor-Action-Agreement)**:
| 요소 | 질문 | 예시 |
|------|------|------|
| Actor | 누가? | @철수 |
| Action | 무엇을? | 계획 수립 |
| Agreement | 언제까지? | 다음 주까지 |

---

### 3. 시간 정보

```
회의록에는 반드시 다음 정보가 있어야 합니다:
- 회의 일자: 언제 있었나?
- 참여자: 누가 있었나?
- 예상 소요 시간 / 실제 소요 시간: 얼마나 했나?
```

**근거**: [Expert] 조직 학습 관점 — 맥락 이해에 필수

## 회의 전 준비

### 회의 전 체크리스트

```
□ 회의 목적이 명확한가?
□ 참여자 목록이 확정되었는가?
□ 이전 회의록/관련 문서를 확인했는가?
□ 회의 템플릿이 준비되었는가?
□ 회의 중에 빠르게 기록할 수 있는 노트 환경인가?
```

### 사전 준비가 중요한 이유

**[Moderate] 연구에 따르면**, 사전 준비가 된 회의는:
- 결정 도달 시간이 30% 단축
- 회의 참여자 만족도가 25% 향상
- 액션 아이템 실행률이 40% 증가

## 회의 중 기록

### 회의 중 기록 원칙

| 원칙 | 설명 |
|------|------|
| **실시간 분류** | 발언을 들으면서 "결정/논의/액션"으로 구분 |
| **요약 기록** | 모든 발언이 아니라 핵심만 기록 |
| **질문 기록** | "이 회의의 핵심 질문은 무엇인가?"를 먼저 적기 |
| **인용 표시** | 중요한 발언은 인용 부호로 표시 |

### 회의 중 노트템플릿

```
회의: [주제] | 일자: MM/DD | 참여: @A, @B, @C

[핵심 질문 1]
- 결정: ...
- 액션: @A가 ...

[핵심 질문 2]
- 결정: ...
- 액션: @B가 ...

기타 발언:
- ...
```

### 회의 중 팁

1. **"이것은 결정인가, 논의인가?"를 계속 자문**
2. **액션 아이템이 나오면 즉시 "누가/무엇을/언제까지" 확인**
3. **논의의 흐름을 타면 화살표나 숫자로 흐름 표시**
4. **이해가 안 되면 즉시 질문 (회의록 질문용!)**

## 회의 후 정리

### 회의 후 24시간 내 정리

**근거**: [Strong] Ebbinghaus Forgetting Curve — 24시간 내에 70% 기억 소실

### 정리 프로세스

```
1단계: 노트 정리 (10분)
   └─ 전체 내용 훑으며 흐름 파악

2단계: 섹션 분리 (20분)
   ├─ 결정 사항 추출
   ├─ 논의 요약 작성
   └─ 액션 아이템 확정

3단계: 맥락 보강 (15분)
   └─ 결정의 이유, 논의의 배경 추가

4단계: 검토 (5분)
   └─ 불참자도 이해할 수 있는지 확인

총: 약 50분
```

### 섹션별 작성 팁

#### 📌 핵심 결정 (Decisions)

```
작성 포맷:
[명확한 결정] — (이유/배경)

예시:
[A 방안 채택] — (이유: 비용이 20% 절감되고 구현 기간도 단축됨)
```

**작성 원칙**:
- "우리는 ~하기로 했다"가 아니라 "~ 채택", "~ 도입" 등 명확한 동사 사용
- 이유가 명확해야 함 (없으면 회의 중에 확인!)
- 동의자가 누구인지 표시 (분열된 결정일 때 유용)

---

#### 💡 논의 요약 (Discussion Summary)

```
작성 포맷:
주장 → 반론 → 결론

예시:
[철수] A 방안이 비용 효율적임
   ↳ [영희] 하지만 B 방안이 기술적으로 더 안정적임
      ↳ [민수] A 방안을 채택하되, 안정성 테스트 추가하자
         → 결정: A 방안 채택 + 안정성 테스트 병행
```

**작성 원칙**:
- 모든 발언이 아니라 핵심 턴닝 포인트만 기록
- 흐름이 화살표로 명확히 보이도록
- 결론이 논의에서 자연스럽게 도출되도록

**[Contested] 통념 경고**:
> "모든 토론 과정을 기록해야 한다"는 오해가 있습니다.
> 연구에 따르면 대부분의 비필요 대화는 노이즈로 작용하며, 검색을 방해합니다.

---

#### ✅ 실행 사항 (Action Items)

```
작성 포맷:
@담당자 — 내용 — 마감일 — 우선순위

예시:
@철수 — 구체 계획 수립 — 03/22 — 🔴높음
```

**작성 원칙**:
- 담당자 없으면 아예 등록하지 않기
- 마감일이 없으면 "TBD"라고 명시하고 ASAP로 표시
- 우선순위 표시 (🔴/🟡/🟢)

**AAA 트리플 체크리스트**:
- [ ] Actor: 누가?
- [ ] Action: 무엇을?
- [ ] Agreement: 언제까지?

## 자주 하는 실수

### 실습 1: 모든 것을 기록하려 함

```
❌ 잘못된 방식:
철수: "음... 아 그래서 제 생각에는 이 방향이 괜찮을 것 같은데,
       다만 여기서 고려해야 할 점들이 몇 가지 있어요..."

✅ 올바른 방식:
[철수] A 방향 괜찮음 (단, 고려할 점 있음)
```

**근거**: [Strong] Cognitive Load Theory — 정보가 많을수록 이해도는 떨어짐

---

### 실습 2: 결정과 논의가 섞임

```
❌ 잘못된 방식:
우리는 A 방안을 채택하기로 했다. 왜냐하면 철수가 비용이 절감된다고 했고,
영희도 동의했고, 민수도 긍정적이었기 때문이다.

✅ 올바른 방식:
[결정] A 방안 채택
[논의] 비용 절감 (철수), 기술적 안정성 (영희), 시간 단축 (민수)
```

---

### 실습 3: 액션 아이템에 담당자 없음

```
❌ 잘못된 방식:
- 다음 주까지 계획 수립
- 보고서 작성

✅ 올바른 방식:
- @철수가 다음 주까지 계획 수립
- @영희가 보고서 작성
```

**근거**: [Expert] David Allen — 담당자가 없으면 실행되지 않음

---

### 실습 4: 맥락 없는 기록

```
❌ 잘못된 방식:
결정: A 방안 채택
액션: @철수가 구현

✅ 올바른 방식:
결정: A 방안 채택 (이유: 비용 20% 절감 + 구현 기간 2주 단축)
액션: @철수가 A 방안 구현 (마감: 03/22)
```

## 효과적인 공유

### 공유 타이밍

| 타이밍 | 효과 |
|--------|------|
| 회의 직후 (1시간 이내) | 참여자 기억이 생생함 |
| 회의 후 24시간 이내 | 최적 (망각 방지) |
| 그 이후 | 효과 급격히 감소 |

### 공유 방법

```
1. 회의록 공유 이메일/메시지 예시:

제목: [회의록] [회의 주제] (MM/DD)

안녕하세요,
오늘 [회의 주제]에 대한 회의록입니다.

📌 핵심 결정:
- A 방안 채택 (이유: 비용 절감)
- 다음 회의일: 03/22

✅ 실행 사항:
- @철수: 계획 수립 (03/22까지)
- @영희: 보고서 작성 (03/22까지)

상세 내용은 첨부파일을 확인해 주세요.
피드백 있으시면 MM/DD까지 부탁드립니다.
```

### 피드백 수집

```
회의록에 대한 피드백 질문:

1. "이 회의록에서 핵심을 파악하는 데 걸리는 시간은?"
2. "무엇이 가장 유용했나요?"
3. "무엇이 불편했나요?"
4. "추가/수정할 부분이 있나요?"
```

## 요약: 회의록 작성의 핵심

1. **Core Practice 3가지를 지키기**
   - 섹션 분리 / 행위자 명시 / 시간 정보

2. **구조가 중요하다**
   - 길이가 아니라 구조의 명확성

3. **회의 중에 분류하며 기록**
   - 결정/논의/액션 실시간 구분

4. **24시간 내에 정리하고 공유**
   - 망각 곡서 이전에 정리

5. **불참자도 이해할 수 있는가?**
   - 최종 체크 포인트
