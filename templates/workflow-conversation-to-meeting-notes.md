## 개요

### 워크플로우 목표

```
[전체 대화 텍스트]  →  [구조화된 회의록]
   5,000자+              1,000-1,500자
   혼란 스러움            명확한 구조
```

### 변환의 핵심 원칙

| 원칙 | 설명 |
|------|------|
| **구조화** | 결정/논의/액션 섹션 분리 |
| **압축** | 20-30% 압축률 유지 (핵심 맥락 보존) |
| **명확성** | 누가/무엇을/언제까지 트리플 추출 |
| **맥락** | 결정의 이유, 논의의 흐름 보존 |

## 상세 프로세스

---

### 단계 1: 소스 분석 (Source Analysis)

**목표**: 전체 대화 내용의 구조와 흐름 파악

#### 수행 항목

```
1. 전체 대화 읽기 (한 번에)
   └─ 대략적인 흐름 파악

2. 회의 기본 정보 추출
   ├─ 회의 주제/목적
   ├─ 참여자 식별 (이름/역할)
   ├─ 시간 정보 (날짜, 소요 시간)
   └─ 회의 유형 (연구/실행/아이디어 등)

3. 핵심 질문 식별
   └─ "이 회의의 핵심 질문은 무엇인가?"
```

#### 체크리스트

- [ ] 회의 주제가 명확한가?
- [ ] 참여자가 모두 식별되었는가?
- [ ] 회의 유형이 파악되었는가?
- [ ] 1-3개의 핵심 질문이 추출되었는가?

---

### 단계 2: 섹션 추출 (Section Extraction)

**목표**: 대화 내용을 결정/논의/액션으로 분류

#### 추출 전략

| 섹션 | 추출 키워드 | 예시 표현 |
|------|-------------|-----------|
| **결정** | 결정, 채택, 하기로, 확정 | "A 방안 채택", "이걸로 하자" |
| **논의** | ~하는데, ~지만, 고려 | "A가 좋지만 B도 있어" |
| **액션** | ~하겠다, ~할게, 담당 | "내가 할게", "철수가 할 거야" |

#### 작업 프로세스

```
1. 전체 대화를 한 번에 읽으며 마킹
   └─ 결정: [D], 논의: [Dsc], 액션: [A]

2. 각 마킹된 내용을 카피하여 별도 메모에 정리
   └─ 각 섹션별로 그룹화

3. 중복/불필요한 내용 제거
   └─ 같은 내용이 반복되면 하나로 통합
```

#### 예시

```
[소스 텍스트]
철수: "음... A 방안이 비용 효율적일 것 같아."
영희: "그렇네. 근데 B 방안도 안정적이야."
철수: "그러면 A 방안 채택하자."
영희: "좋아. 그럼 내가 A 방안 구현할게."

[마킹 후]
철수: [Dsc] A 방안이 비용 효율적
영희: [Dsc] B 방안도 안정적
철수: [D] A 방안 채택
영희: [A] A 방안 구현 (@영희)
```

---

### 단계 3: 구조화 (Structuring)

**목표**: 추출된 내용을 회의록 템플릿에 맞게 정리

#### 3-1: 결정 사항 (Decisions)

```
[작성 포맷]
[결정] 명확한 결정 내용 — (이유/배경)

[소스 → 구조화 예시]
[소스]
철수: "A 방안 채택하자. 비용이 절감되니까."

[구조화]
[D-1] A 방안 채택 — (이유: 비용 절감)
```

**작성 팁**:
- 불확실한 표현 제거 (~인 것 같아 → 채택)
- 이유가 명확하지 않으면 소스에서 찾아 추가
- 결정이 없으면 "결정 없음"으로 명시

---

#### 3-2: 논의 요약 (Discussion Summary)

```
[작성 포맷]
주장 → 반론 → 결론

[소스 → 구조화 예시]
[소스]
철수: "A 방안이 비용 효율적일 것 같아."
영희: "그렇네. 근데 B 방안도 안정적이야."
철수: "그러면 A 방안 채택하자."

[구조화]
[논의] A/B 방안 비교
   [철수] A 방안: 비용 효율적
      ↳ [영희] B 방안: 안정적
         → 결론: A 방안 채택
```

**작성 팁**:
- 발언자 표시: @이름 또는 [이름]
- 흐름을 화살표로 명확히
- 결론은 결정 사항과 연결

---

#### 3-3: 액션 아이템 (Action Items)

```
[작성 포맷]
@담당자 — 내용 — 마감일 — 우선순위

[소스 → 구조화 예시]
[소스]
영희: "좋아. 그럼 내가 A 방안 구현할게. 다음 주까지."

[구조화]
@영희 — A 방안 구현 — 다음 주(03/22) — 🔴높음
```

**AAA 트리플 체크**:
- [ ] Actor: 누가? (담당자)
- [ ] Action: 무엇을? (구체적 내용)
- [ ] Agreement: 언제까지? (마감일)

**작성 팁**:
- 담당자가 없으면 "TBD"로 표시하고 ASAP 확인
- 마감일이 없으면 "TBD"로 표시
- 우선순위 표시 (🔴높음 / 🟡중간 / 🟢낮음)

---

### 단계 4: 맥락 보강 (Context Enhancement)

**목표**: 추출된 정보에 맥락 추가

#### 보강 항목

| 섹션 | 보강할 맥락 |
|------|-------------|
| **결정** | 이유/배경, 동의자, 이견 여부 |
| **논의** | 참여자의 역할, 논의의 깊이 |
| **액션** | 관련 결정, 선행 조건, 의존 관계 |

#### 작업 프로세스

```
1. 결정 사항의 이유/배경 확인
   └─ 소스에서 결정 이유 찾기
   └─ 없으면 "이유: 명시되지 않음"으로 표시

2. 논의의 참여자와 역할 확인
   └─ 누가 어떤 입장을 취했는지

3. 액션 아이템의 관련 결정 연결
   └─ 이 액션이 어떤 결정에서 나왔는지
```

---

### 단계 5: 검토 완료 (Review & Finalize)

**목표**: 불참자도 이해할 수 있는지 최종 확인

#### 검토 체크리스트

```
[기본 정보]
□ 회의 일자가 있는가?
□ 참여자가 모두 명시되었는가?
□ 회의 목적이 명확한가?

[결정 사항]
□ 모든 결정이 추출되었는가?
□ 각 결정에 이유/배경이 있는가?
□ 결정 사항이 명확한 동사로 표현되었는가?

[논의 요약]
□ 논의의 흐름이 명확한가?
□ 결론이 논의에서 자연스럽게 도출되는가?
□ 발언자가 명시되어 있는가?

[액션 아이템]
□ 모든 액션 아이템이 추출되었는가?
□ 각 액션에 담당자가 있는가?
□ 각 액션에 마감일이 있는가?
□ 우선순위가 표시되었는가?

[전체]
□ 불참자도 이해할 수 있는가?
□ 3분 안에 핵심을 파악할 수 있는가?
□ 불필요한 정보가 제거되었는가?
```

#### 불참자 테스트

```
"회의에 참여하지 않은 사람이 이 회의록을 읽고
다음 질문에 답할 수 있는가?"

1. 무엇이 결정되었는가?
2. 왜 그렇게 결정했는가?
3. 누가 무엇을 해야 하는가?
4. 다음 회의는 언제인가?
```

## 자동화 옵션

### 옵션 1: LLM 활용

```
[프롬프트 예시]

다음 전체 대화 내용을 바탕으로 회의록을 작성해 주세요.

요구사항:
1. 결정 / 논의 / 액션 섹션으로 분리
2. 액션 아이템에 @담당자 - 내용 - 마감일 포함
3. 논의는 주장-반론-결론 구조로 요약
4. 불필요한 내용은 제거하고 핵심만 포함

[대화 내용]
{전체 대화 텍스트 붙여넣기}
```

**Human-in-the-Loop**:
1. LLM으로 초안 생성
2. 인간이 검토 및 수정
3. 최종 확정

**근거**: [Emerging] Zhang et al. (2023) — LLM 기반 요약 + 인간 검수가 순수 LLM보다 더 효과적

### 옵션 2: 정규 표현식 활용

```
[결정 추출 패턴]
/(.+?)(?:채택|하기로|황정|결정)/g

[액션 아이템 추출 패턴]
/(.+?)가 (.+?) 할게/g

[마감일 추출 패턴]
/(다음\s*주|내일|오늘|(\d{1,2})\/(\d{1,2}))/g
```

### 옵션 3: 자동화 스크립트 (Python 예시)

```python
import re

def extract_decisions(text):
    """결정 사항 추출"""
    pattern = r'(.+?)(?:채택|하기로|황정|결정)'
    return re.findall(pattern, text)

def extract_actions(text):
    """액션 아이템 추출 (AAA 트리플)"""
    # Actor 추출
    actors = re.findall(r'(\w+)(?:가|이)\s(.+?)\s(할게|할거야)', text)
    return actors

def extract_dates(text):
    """마감일 추출"""
    dates = re.findall(r'(다음\s*주|내일|오늘|(\d{1,2})\/(\d{1,2}))', text)
    return dates

def create_meeting_notes(text):
    """전체 워크플로우 실행"""
    decisions = extract_decisions(text)
    actions = extract_actions(text)
    dates = extract_dates(text)

    # ...
```

## 예시: 전체 대화 → 회의록 변환

### 입력 (전체 대화)

```
[회의: 새로운 연구 프로젝트 방향 논의]
일자: 2024-03-15
참여: 철수, 영희, 민수

철수: "자, 오늘은 새로운 연구 프로젝트 방향을 정해야 해.
     두 가지 옵션이 있는데, 하나는 NLP 기반 텍스트 요약,
     다른 하나는 컴퓨터 비전 기반 이미지 분석이야."

영희: "음... 텍스트 요약은 이미 많이 연구됐잖아.
     우리가 새로운 contribution을 만들 수 있을까?"

민수: "그렇지만 우리 팀의 강점이 NLP쪽이잖아.
     이미 데이터도 많이 모여 있고."

영희: "그렇네... 근데 이미지 분석도 재미있을 것 같아.
     최근에 attention mechanism이 이미지 분야에서도
     좋은 성과를 내고 있어서."

철수: "음, 기여도 측면에서는 이미지 분석이 더 낫겠네.
     그리고 우리도 새로운 것을 도전해 보는 게 좋겠어."

민수: "그렇다면 이미지 분석으로 가자.
     그럼 나가 먼저 관련 논문 조사해볼게."

영희: "내가 데이터셋을 찾아볼게.
     ImageNet 외에 다른 데이터셋도 보고."

철수: "좋아. 그럼 이미지 분석으로 결정!
     다음 주 회의 때 각자 준비한 것 공유하자."
```
